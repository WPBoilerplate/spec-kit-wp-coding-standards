# WP Coding Standards — Apply Command

You are running `speckit.wp-coding-standards.apply`.

This command takes violations from the most recent `review` or `violation-detection` output and converts them into structured, prioritized fix tasks. It injects those tasks into `specs/<feature>/tasks.md` so the AI has explicit guidance to resolve standards violations during implementation.

---

## Operating Constraints

- Read the most recent review output from the current conversation context, or ask the user to paste it.
- Do NOT re-run the review. Use violations already identified.
- Inject tasks into `tasks.md` — do not create a separate file.
- Do not mark any task as complete. All injected tasks are `[ ]` pending.
- Preserve existing tasks. Append a clearly labelled `## WP Standards Fix Tasks` section.

---

## Step 1 — Load Violations

Read violations from the most recent `review` or `violation-detection` output in context.

If no violations are available, ask:

```text
No violation report found in context. Please run /speckit.wp-coding-standards.review first,
or paste the violations table here.
```

---

## Step 2 — Map Violations to Priority

| Violation Severity | Task Priority |
|---|---|
| CRITICAL | P0 — must fix before merge |
| HIGH | P1 — should fix before release |
| MEDIUM | P2 — safe as tracked debt |
| LOW | P3 — opportunistic cleanup |

---

## Step 3 — Generate Fix Tasks

For each violation, generate a fix task using this structure:

```markdown
- [ ] [P{priority}] [{Rule}] Fix: {plain-English description of what to change}
  - File: {path}:{line}
  - Rule: {WPCS rule name}
  - Fix: {specific corrective action}
```

### Fix Guidance by Rule

**EscapeOutput**
```
Wrap the output in the most specific escaping function:
- Text content → esc_html()
- HTML attributes → esc_attr()
- URLs → esc_url()
- JavaScript → esc_js()
- Translated strings → esc_html__() / esc_attr__()
```

**NonceVerification**
```
Add nonce verification before the superglobal read:
- AJAX handlers → check_ajax_referer( 'action_name', 'nonce_field' )
- Form submissions → wp_verify_nonce( $_POST['_wpnonce'], 'action_name' )
```

**ValidatedSanitizedInput**
```
Sanitize using the most specific function:
- Plain text → sanitize_text_field()
- Integer → absint() or intval()
- Email → sanitize_email()
- URL → esc_url_raw()
- HTML content → wp_kses_post()
- Textarea → sanitize_textarea_field()
```

**PreparedSQL**
```
Use $wpdb->prepare() with typed placeholders:
- String → %s
- Integer → %d
- Float → %f
Never use sprintf() or string concatenation to build queries.
```

**PrefixAllGlobals**
```
Rename to include the project prefix:
- Functions: my_plugin_function_name()
- Constants: MY_PLUGIN_CONSTANT_NAME
- Hooks: my_plugin_hook_name
- Classes: My_Plugin_Class_Name
```

**DeprecatedFunctions**
```
Replace with the current alternative documented in the WordPress developer handbook.
Check https://developer.wordpress.org/reference/ for the replacement.
```

**Capabilities**
```
Add current_user_can() check before data access or mutation:
- Admin operations → current_user_can( 'manage_options' )
- Post editing → current_user_can( 'edit_post', $post_id )
- Custom capability → current_user_can( 'my_plugin_manage_settings' )
```

---

## Step 4 — Inject into tasks.md

Locate `specs/<feature>/tasks.md`. If it does not exist, ask the user for the correct path.

Append to the end of the file:

```markdown
## WP Standards Fix Tasks

> Generated by /speckit.wp-coding-standards.apply — [date]
> Profile: [profile] | Violations: [count]

### P0 — Critical (must fix before merge)
- [ ] [P0] [EscapeOutput] ...

### P1 — High (fix before release)
- [ ] [P1] [PrefixAllGlobals] ...

### P2 — Medium (tracked debt)
- [ ] [P2] [SlowDBQuery] ...

### P3 — Low (opportunistic)
- [ ] [P3] [YodaConditions] ...
```

Omit sections that have no violations at that priority level.
